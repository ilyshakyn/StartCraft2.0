Что реализовано:
Пользователь в главном меню выбирает количество дронов через ползунок.

Значение сохраняется в синглтоне GameSettings, который не уничтожается при смене сцены.

В игровой сцене DroneSpawnManager включает нужное количество дронов из пула (остальные удаляются).

Все дроны заранее находятся в сцене, но выключены.

Дроны работают по состояниям: ищут ресурс, собирают, доставляют.

При доставке вызывается событие OnResourceDropped, на которое подписан скрипт DropEffectPlayer — он запускает эффект.

Подписка на событие происходит только при подборе ресурса, отписка — после срабатывания.

Используемые паттерны и логика:
Singleton — GameSettings и менеджеры (как GameManager)

Event (Observer) — DroneEventChannel для уведомлений о выгрузке ресурса

FSM (Finite State Machine) — система состояний для поведения дронов

DI (через ссылки в инспекторе) — дроны не жёстко зависят от эффектов или менеджеров
